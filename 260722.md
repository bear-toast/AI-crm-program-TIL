기술을 왜 도입하게 되는지를 고려해야 함

#### 기획
- 문제 정의 -> 목표 설정 -> 액션 플랜
- 단계
	- 문제 정의
	- 고객 분석
	- 기능 우선순위 결정
	- 커뮤니케이션
	- 단계별 개선 및 피드백
- 주제
	- 내가 불편
	- 주변 사람 불편
	- 있으면 좋은 것보단 꼭 해결해야 하는 것
- 문제 발견 -> 고객 이해 -> 유사서비스 확인 -> 해결 방법 설계 -> 기술 선택 -> 개발
- bad
	- 광범위, 대상 불명확, 시장 차별성 x, 기술 스택 우선
- good
	- 구체적 고객, 구체적 문제, 작고 뾰족 to 확장성, 많은 기능의 맥가이버칼 아닌 하나의 기능 완벽 해결
- 새로운 제품 출시 cycle
	- 아이디어 관리
	- 구체화 및 검증
	- 사업성 분석 및 마케팅 전략 수립
	- 제품 서비스 개발
	- UI 정의, 매뉴얼 작성
	- 출시 및 홍보
- Working backward - Beachhead strategy
	- 보도자료 작성
	- FAQ 작성
	- UI 정의, 매뉴얼 작성
	- 평가/검증
	- 제품/서비스 개발
- 단계들
	- Listen - Persona 만들기
		- 많이 구체적
		- 스테레오타입 아닌 특별한 고객
		- 현실적으로 존재할 법한 사람
		- 해결하려고 고민하고 애쓰는 고객
	- Define - Today Statement
		- 꼭 해결할 문제
		- like 일기
